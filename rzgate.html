RZGate: Instant HTTP Access to Roomzin

A lightweight, high-performance HTTP/JSON proxy that lets any system tap into Roomzin's sub-millisecond inventory engine — without native SDK integration.

Many teams love Roomzin's speed but work with existing HTTP-based platforms or legacy booking engines that aren't ready for a native SDK yet. RZGate is our lightweight HTTP/JSON proxy that makes adoption instant and risk-free.

Run RZGate

1. Get the binary

Request the single-file executable from the Roomzin support team (Linux x86_64, 3 MB). No dependencies, no package manager, no Docker.

2. Start the gateway

./rzgate --https true

RZGate expects TLS certificates and will exit if they are missing when https enabled.

3. Optional flags

Flag Meaning Default
-c, --config Path to rzgate.yml ./rzgate.yml → /etc/rzgate/rzgate.yml
--auth_enabled Enables authentication/access control false
--http Enables http protocol access false
--https Enables https protocol access false
-t, --tokens_path Path to auth.yml ./auth.yml → /etc/rzgate/auth.yml
--tls-cert PEM certificate file value from rzgate.yml
--tls-key PEM private-key file value from rzgate.yml

4. Required files

  • rzgate.yml – port, Roomzin cluster endpoints, TLS paths, log level
  • auth.yml – full-access and read-only bearer tokens (auto-reloaded)
  • cert.pem / key.pem – TLS assets (no plain-HTTP mode)

Configuration file (rzgate.yml)

A single YAML file controls every runtime setting. Keys are optional unless marked required; defaults are shown.

Key Purpose Default / Example
auth_enabled Enables auth/access control check for RZGate false
http_enabled Enables http access to RZGate false
https_enabled Enables https access to RZGate false
http_port Port RZGate listens on for http 8777
https_port Port RZGate listens on for https 3443
roomzin_api_port Roomzin cluster HTTP health/portfolio port 8080
roomzin_tcp_port Roomzin native TCP protocol port 7777
seed_hosts Comma-separated list of initial Roomzin nodes (no port) "roomzin-0,roomzin-1,roomzin-2"
tokens_path Path to bearer-token file (auto-reloaded) "./auth.yml"
tls_cert_path PEM certificate for HTTPS "./certs/cert.pem"
tls_key_path PEM private key "./certs/key.pem"
Tip: seed_hosts are host-names only—ports are taken from roomzin_api_port and roomzin_tcp_port. RZGate discovers the rest of the cluster automatically.

Auth file (auth.yml)

Bearer tokens are loaded once at start-up and watched for changes—no restart required.

# Token RZGate itself uses to talk to the Roomzin cluster
roomzin_token: "abc123"

# Client tokens that can read AND write
full_access_tokens:
  - "rzgate123"
  - "rzgate456"

# Client tokens that can only query/search
read_only_tokens:
  - "partner789"
  - "externalABC"
  • roomzin_token must match the token configured on your Roomzin nodes.
  • Any string is valid; rotate by editing the file—RZGate reloads within seconds.
  • Returning 401 Unauthorized? The supplied bearer token is missing or not listed above.

Metrics

A single /metrics endpoint exports everything Prometheus needs—no extra exporters required. The endpoint is protected by the same bearer-token auth as every other route; any valid token (read-only or full-access) will work.

Exported series

  • api_commands_total – total JSON commands processed by RZGate
  • api_bytes_received_total – HTTP request body bytes
  • api_bytes_sent_total – HTTP response body bytes
  • api_client_errors_total – 4xx / 5xx responses sent
  • api_client_login_fail_total – rejected bearer tokens
  • backend_followers_total – live Roomzin follower nodes RZGate is connected to (gauge)
  • backend_leader_change_total – leader elections detected since startup
  • backend_disconnect_total – TCP disconnects from any Roomzin node
Ops note: RZGate does not restrict access to /metrics by IP or token type. If you want to hide it from the internet, block the path at your load balancer, firewall, or Prometheus-agent sidecar.
